# 监控与应急运行手册

## 指标与阈值（示例）
- 性能：p50/p95/p99 延迟、吞吐、CPU/Mem、GC
- 稳定性：错误率、5xx、超时、重试/熔断触发次数
- 业务：关键成功率、转化率（如有）、实验成功率
- 反馈：用户工单数、情感倾向

## 告警分级
- P1：持续错误率>阈值 或 p99 激增且无法自愈 → 立即回滚
- P2：性能退化但可自愈/降级 → 观察+限流/熔断
- P3：非核心告警 → 记录并排期

## 操作流程
1. 确认告警来源与范围，检查最近发布/配置变更
2. 查看 Dashboard：延迟、错误率、资源、长尾
3. 若 P1，执行分钟级回滚（见 release-switch-manual）
4. 若 P2，启用限流/熔断/降级，收集诊断数据
5. 事件记录：`rollbacks` 或事件表，触发 `tech.baseline.update`

## 可观测性工具链（默认）
- Prometheus/Grafana：指标与告警
- Loki/ELK：日志检索
- Tempo/Jaeger：分布式追踪
- Alertmanager：告警路由与静默
