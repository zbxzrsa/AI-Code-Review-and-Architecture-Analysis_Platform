# 技术决策日志模板

## 记录字段
- 候选名称/来源：外部技术、模型、库或架构策略
- 提交来源：v3 监测任务 ID、参考文献/版本
- 评估参数：性能/资源/兼容/安全/可运维性
- 基线对比：与当前生产基线差异、提升百分比或风险点
- 结论：接受/暂缓/拒绝 + 理由
- 路径：进入 v1 实验的计划、里程碑、A/B 或 Shadow 方案
- 风险与缓解：潜在破坏性变更、迁移策略
- 审计：签名（v1/v3/v2）、时间戳

## 示例
- 候选：编排引擎升级 + 新模型安全审查器
- 提交：tech.candidate#2025-xx-yy
- 评估：吞吐 +22%，p99 -15%，兼容性风险低；需新增 ACL
- 结论：接受，进入 v1 三周期；Shadow + A/B
- 风险：ACL 漏配 → 强制策略检查；回滚使用旧镜像 + 旧策略
